## `agentix build`

Package one project root into a deploy-ready sandbox image. Plugins
arrive transitively via the project's `[project].dependencies` — the
CLI never enumerates them.

```bash
agentix build                                # current directory's pyproject
agentix build path/to/project                # explicit project root
agentix build . -o my-agent:0.1.0            # explicit image tag
agentix build . --dry-run                    # stage Dockerfile, no docker invoke
agentix build . --runtime-image agentix/runtime:0.2.0   # different base
```

The positional arg is a path to a directory containing `pyproject.toml`.
Whatever that project's `[project].dependencies` resolves to via pip
ends up in the bundle's `/nix/runtime/` venv; at runtime the framework
imports target modules by their Python path on first remote call.
Multi-plugin bundles are expressed by declaring multiple `agentix-*`
dists as deps:

```toml
# my-agent/pyproject.toml
[project]
name = "my-agent"
version = "0.1.0"
dependencies = [
    "agentixx>=0.1.0",
    "agentix-runtime-basic>=0.1.0",  # bash + files
    "agentix-claude-code>=0.1.0",    # claude CLI
    "agentix-swebench>=0.1.0",       # scoring
]
```

Then `agentix build` from the project dir builds the multi-plugin image.

`-o / --output / --tag` overrides the default `<name>:<version>` image
tag from the pyproject. `--dry-run` stages the Dockerfile to
`./build/<short>/` and exits without invoking docker.

## `agentix deploy`

Provision a sandbox.

```bash
agentix deploy <backend> --image <tag>
agentix deploy local --image my-agent:0.1.0
agentix deploy local --image my-agent:0.1.0 --detach
agentix deploy daytona --image docker.io/me/my-agent:0.1.0    # currently NotImplementedError
```

`<backend>` is any registered `agentix.deployment` plugin. The local
Docker backend ships in `agentix-deployment-docker`; Daytona, E2B, and
third-party backends ship as their own packages.

Foreground by default: prints `runtime_url`, waits for Ctrl-C, tears
the sandbox down. `--detach` exits after `create()` and prints the
handle.

## Configuration

Most config reads from environment variables.

| Variable                          | Used by                    | Purpose                                |
| --------------------------------- | -------------------------- | -------------------------------------- |
| `AGENTIX_BIND_PORT`               | runtime server             | Bind port (default 8000)               |
| `AGENTIX_UPLOAD_ROOT`             | `agentix.files` module     | Sandbox-side root for file I/O         |
| `DAYTONA_API_KEY`                 | `daytona` deployment       | API auth                               |
| `E2B_API_KEY` / `E2B_TEMPLATE_ID` | `e2b` deployment           | API auth + template                    |

Third parties that want their own top-level verb should ship a
separate `console_scripts` binary (`agentix-yourcmd`) rather than
registering against the central CLI.
